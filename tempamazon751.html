Amazon Interview | Set 110 (On-Campus)

  • Difficulty Level :\nExpert
  • Last Updated :\n07 May, 2021

Hi, Amazon visited our campus few days back. Before I start off with my experience, here are few suggestions.\xc2\xa0

1. Don\xe2\x80\x99t just go through the codes. First try it on your own, that\xe2\x80\x99s very important. Once u have cracked a problem, write it in a paper and check for the corner cases.\xc2\xa0

2. For the online round. GeeksQuiz is enough to get you through the MCQs. For the coding part, once u have cracked a problem, always try to run the program at one go.\xc2\xa0

3. OS, DBMS and Networking. If you get a good hold on these subjects, it will make you stand apart from other candidates.\xc2\xa0

First day:\xc2\xa0
20 MCQs(Data structure, C, C++, OS, Probability).\xc2\xa0
2 programs :\xc2\xa0

1. Given an array of integers. Segregate all the non-zero numbers at the beginning. Print the number of non-zero integers and the minimum number of swaps required for these operations.\xc2\xa0

Eg.  : I/p : 1, 0, 0, -6, 2, 0\n         o/p : Number of non-zero integers : 3\n                   Minimum number of swaps : 2

2. Given a text txt[0..n-1] and a pattern pat[0..m-1], write a function search(char pat[], char txt[]) that prints all occurrences of pat[] and its permutations (or anagrams) in txt[]. You may assume that n > m.\xc2\xa0

E.g. : I/p : txt[] = \xe2\x80\x9cBACDGABCDA\xe2\x80\x9d    pat[] = \xe2\x80\x9cABCD\xe2\x80\x9d\n          o/p : Found at index 0\n                    Found at index 5 \n                    Found at index 6

Second day:\xc2\xa0
Suggestion: Always think loud when you are coding\xc2\xa0
Round 1 : PI 45 min\xc2\xa0
1. Discussion on Project.\xc2\xa0
2. Code : LCA of a BST.(check for all corner cases)\xc2\xa0
3. Code : LCA of a Binary Tree(check for corner cases)\xc2\xa0
4. Code : Maximum sum subarry along with its starting and ending index.\xc2\xa0
5. Code : Heap operations\xc2\xa0

Round 2 : PI (With Manager) 1 hr 15 min\xc2\xa0
1. Discussion on project.\xc2\xa0
2. Code : Given an array : A1[] = 2, 1, 2, 5, 7, 1, 9, 3, 6, 8, 8\xc2\xa0
A2[] = 2, 1, 8, 3\xc2\xa0
Sort A1 in such a way that the relative order among the elements will be same as those are in A2. If the element is not present in A2, append them at last in sorted order.\xc2\xa0
o/p : 2, 2, 1, 1, 8, 8, 3, 5, 6, 7, 9\xc2\xa0

As I carried on telling him the approach, he went on modifying the question ( what if A2 is much larger than A1 ?, what if they are of the same size ?)\xc2\xa0

A lot of discussion on complexities for all modifications. He was quite happy with my answers.\xc2\xa0
Finally he asked me to code the O(nlogn) approach.\xc2\xa0

Round 3 (Subject Round) 1 hr\xc2\xa0
1. OS : Virtual Memory (lot of discussion of its advantages, thrashing, etc)\xc2\xa0
Then he asked me design a LRU replacement policy(approach) which i did. Then he asked to implement FIFO replacement policy with stacks. I did it using two stacks.\xc2\xa0

2. Networking : He started with Transport Layer(discussion on ports. How data from applications are changed into packets ? etc).\xc2\xa0
Then he went to Network Layer (A lot of discussion on subnet masking and how it is used by routers to map the IP address of incoming packet, how a Subnet is different from Internet).\xc2\xa0
Next we moved to Data Link Layer(Discussion MAC addresses, sliding window protocols, etc).\xc2\xa0

3. DBMS: What is indexing?, Why indexing ?, How to implement indexing ?, What are the different types of Index? .\xc2\xa0
Differences among Inner Join, Left Outer Join, and Right Outer Join. He followed it up with a SQL query on Outer Join.\xc2\xa0

As my first three rounds went very well, fourth round was of 15 minutes only.\xc2\xa0

Round 4 (Bar Raiser) 15 min.\xc2\xa0
1. Logical question. Given a prime number \xe2\x80\x98p\xe2\x80\x99. Prove that p^2-1 is always divisible by 24.\xc2\xa0
2. Find 2nd maximum element in a given array in one iteration.\xc2\xa0

All the best !\xc2\xa0
