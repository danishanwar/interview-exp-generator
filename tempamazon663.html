Amazon Interview Experience | 194 (For Software Support Engineer)

  • Difficulty Level :\nEasy
  • Last Updated :\n28 Jun, 2019

It was a usual Job Application at Amazon\xe2\x80\x99s Job Site, I got their call few days later.

Process:- 1 Written Aptitude(Coding) + 3 Technical Round + 1 HR Round

1) One Written Test \xe2\x80\x93 8 Problem statement \xe2\x80\x93 1 Hour Time \xe2\x80\x93 from Basic to Mid-level difficulty, for which we have to write the Logic/Algorithms
Some which I remember were good enough are \xe2\x80\x93
i) Given a Folder path in a Directory Structure. E.g. \xe2\x80\x9ctemp/reg\xe2\x80\x9d; you need to find all the files with extensions \xe2\x80\x9c.c\xe2\x80\x9d in this folder as well as in the sub-folders.
ii) Finding the Longest Palindrome Sub-String in the given string.
iii) From the given string, print the Alphabets which occur more than once in the string.
iv) Given two Integers A & B, Print the number of bits to be flipped in order to convert \xe2\x80\x98A\xe2\x80\x99 to \xe2\x80\x98B\xe2\x80\x99.
v) Implement EnQueue() and DeQueue() of Queue DS using Push() and Pop() of Stack DS. Write Steps and Sequence.

Possible Solutions:- (Logic)
i) Use Linked-List DS to store the Folder Path in a Node. Open the Folder with First Node, filter all the \xe2\x80\x9c.c\xe2\x80\x9d extension files, and add the sub-folders in the linked-list, until all the nodes in the linked-list are explored.
ii) Use \xe2\x80\x98frame\xe2\x80\x99 variable with value starting from 2, increasing till size \xe2\x80\x98n\xe2\x80\x99, and check by shifting this frame by one position, if the sub-string is a Palindrome or not.
iii) Basic Char to ASCII conversion and back to Characters. Create an Array of 26 slots for alphabets. Convert the input string to UpperCase before proceeding.
iv) Convert to Binary Representation and Compare as String Characters. If not same, then increment the counter. Remember, we have to only display the Number of bits flipped.
v) Use Operational Logic for Priority Queue, and for \xe2\x80\x93
\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0a) EnQueue \xe2\x80\x93 Push everything in Stack1.
\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0b) DeQueue \xe2\x80\x93 (Check if Stack1 is Not empty)Pop everything from Stack1 and Push them one by one to Stack2. Then Pop top element from Stack2 as Result. For Next DeQueue do reverse, i.e. from Stack2 to Stack1.

2) 1st Technical Round
i) Write Program to Reverse a Linked List.
ii) Discussion on a Scenario and Debugging/Troubleshooting the error.

3) 2nd Technical Round
i) The similar Debugging/Troubleshooting scenario.
ii) Add the Numbers present in the String and display.
\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0 E.g. \xe2\x80\x9cabc123def-567ghi89jk\xe2\x80\x9d
\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0\xc2\xa0Here take \xe2\x80\x98-\xe2\x80\x98 as negative number symbol.
iii) Find the Pair of Array elements summing up to value \xe2\x80\x98k\xe2\x80\x99, in a Sorted Array. Optimize it from Complexity \xe2\x80\x98N^2\xe2\x80\x99 to \xe2\x80\x98NlogN\xe2\x80\x99 using Binary Search.
4) 3rd Technical + Managerial Round
i) Some OS Related Questions.
ii) UNIX Commands \xe2\x80\x93 Which I couldn\xe2\x80\x99t recall!
iii) Project Experience and Why want to Join Amazon and similar ones.

Though I couldn\xe2\x80\x99t clear the 3rd Round, the overall experience was good.
